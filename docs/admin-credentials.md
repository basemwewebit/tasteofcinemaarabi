# بيانات الدخول للوحة الإدارة (Admin Credentials)

تم إعداد نظام التوثيق الخاص بلوحة التحكم `iron-session` بشكل يدعم مستخدم مدير (Admin) واحد. 

## بيانات الدخول الافتراضية

أثناء إعداد بيئة التطوير محليًا، تم تكوين المتغيرات التالية في ملف `.env.local`:

- **اسم المستخدم (Username):** `admin`
- **كلمة المرور (Password):** `tasteofcinema2026`

يمكنك استخدام هذه البيانات لتسجيل الدخول إلى لوحة التحكم من خلال الرابط:
`http://localhost:3000/admin/login`

---

## كيفية تغيير اسم المستخدم وكلمة المرور

لأسباب أمنية (خاصة عند رفع المشروع على خادم حقيقي Production)، **يُمنع منعًا باتًا** تخزين كلمة المرور كنص صريح. يتم استخدام التشفير عبر `bcrypt`.

لتغيير بيانات الدخول في أي وقت، اتبع الخطوات التالية:

### 1. تغيير اسم المستخدم
مباشرةً في ملف `.env.local` أو عبر إعدادات البيئة في الخادم، قم بتغيير القيمة:
```env
ADMIN_USERNAME=my_new_username
```

### 2. تغيير كلمة المرور
يجب عليك إنشاء Hash (نسخة مشفرة) لكلمة المرور الجديدة. يمكنك استخدام الطرفية (Terminal) لتوليد الـ Hash عبر تنفيذ هذا الأمر (مع استبدال `YOUR_NEW_PASSWORD` بكلمتك الجديدة):

```bash
node -e "const b=require('bcryptjs');console.log(b.hashSync('YOUR_NEW_PASSWORD',12))"
```

سينتج الأمر نصًا مشفرًا يبدأ بـ `$2a$12$` أو `$2b$12$`. 

قم بنسخ هذا النص المشفر والصقه في المتغير `ADMIN_PASSWORD_HASH` داخل ملف `.env.local`. **هام جداً:** يجب وضع الشرطة المائلة للخلف `\` قبل كل علامة `$`، أو وضع القيمة كاملة بين علامتي اقتباس مزدوجتين `""` مع تخطي علامة `$` باستخدام `\`، ليتمكن خادم `Next.js` من قراءتها بشكل صحيح وعدم محاولة استخدامها كمتغيرات داخلية:

```env
# الطريقة الصحيحة:
ADMIN_PASSWORD_HASH="\$2b\$12\$تشفير_كلمة_المرور_الجديدة_هنا"
```

## ملاحظات أمنية إضافية (مفتاح الجلسة)

يعتمد توثيق الجلسات على المفتاح السري `SESSION_SECRET` الموجود أيضًا في ملف `.env.local`. 
- يجب ألا يقل طول هذا المفتاح عن **32 حرفًا**.
- إذا تم تغيير هذا المفتاح في أي وقت، سيتم تسجيل خروج أي مستخدم نشط حاليًا (سيتم إبطال جميع الجلسات القديمة). 

يمكنك توليد مفتاح سري قوي جديد باستخدام هذا الأمر:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
